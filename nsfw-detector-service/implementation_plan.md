## 4. Model Manager (`app/services/model_manager.py`)

Singleton ที่ load ครั้งเดียวใน FastAPI `lifespan`:

```python
class ModelManager:
    def __init__(self) -> None:
        self.image_processor = None
        self.model = None
        self.device = "cpu"
        self._ready = False
        self._lock = threading.Lock()

    def _resolve_device(self) -> str:
        desired = settings.DEVICE or ("cuda" if settings.USE_GPU else "cpu")
        if desired == "cuda" and not torch.cuda.is_available():
            logger.warning("CUDA requested but unavailable → fallback to CPU")
            return "cpu"
        return desired

    def load(self) -> None:
        self.device = self._resolve_device()
        self.image_processor = AutoImageProcessor.from_pretrained(settings.MODEL_NAME)
        self.model = AutoModelForImageClassification.from_pretrained(
            settings.MODEL_NAME,
        ).to(self.device)
        self.model.eval()
        self._ready = True
        logger.info("Model loaded on device=%s", self.device)

    @property
    def ready(self) -> bool: return self._ready

    def predict(self, image: PIL.Image.Image) -> dict:
        if not self._ready: raise ModelNotReadyError()
        inputs = self.image_processor(images=image, return_tensors="pt").to(self.device)
        with torch.no_grad():
            logits = self.model(**inputs).logits
        probs = torch.softmax(logits, dim=-1)[0]
        ...
```

**Implementation notes:**
- ใช้ `.to(device)` แทน `device_map="auto"` เพื่อควบคุมชัดเจน + log fallback
- `predict()` เป็น sync function → เรียกจาก endpoint ผ่าน `await run_in_threadpool(model_manager.predict, image)` เพื่อไม่บล็อก async event loop
- thread safety: `model.eval()` + `no_grad` read-only inference → safe concurrency พื้นฐาน แต่แนะ `asyncio.Semaphore` จำกัด 1 inference concurrent ถ้า GPU memory เป็นห่วง (config `MAX_CONCURRENT_INFERENCES=1`)

@asynccontextmanager
async def lifespan(app: FastAPI):

## 6. Image Processing Pipeline (`app/services/image_processor.py`)

```python
def preprocess(image: PIL.Image.Image) -> PIL.Image.Image:
    img = ImageOps.exif_transpose(image)  # fix orientation
    img = img.convert("RGB")              # webp/RGBA/P → RGB
    if max(img.size) > settings.MAX_DIMENSION:
        ratio = settings.MAX_DIMENSION / max(img.size)
        new_size = (int(img.width*ratio), int(img.height*ratio))
        img = img.resize(new_size, Image.Resampling.LANCZOS)
    return img
```

- อ่านด้วย `PIL.Image.open(io.BytesIO(await file.read()))` (อย่าลืม `await` ก่อน seek)
- ปิด image ด้วย `with`/`close()` เพื่อ free FD
- ตรวจ MIME จาก magic bytes ด้วย `imghdr`/`PIL Image.format` แทนเชื่อ Content-Type header เพื่อกัน spoof

## 7. Endpoints & Error Envelope

### 7.1 Envelope มาตรฐาน

```json
/* success */
{ "success": true, "data": { ... } }
/* error */
{ "success": false, "error": "message", "detail": "optional" }
```

### 7.2 GET `/health` (public)

```json
{ "success": true, "data": { "status": "ready", "model_loaded": true, "device": "cuda" } }
```

- สถานะ: `ready` / `not_ready` (model_manager.ready)
- ตอบ 503 หาก model ยัง load ไม่เสร็จ แต่ upstream health check มักต้องการ 200 → แนะให้ตอบ 200 โดยมี `model_loaded: false` (รอท่านตัดสินใจ: 200 vs 503)

### 7.3 POST `/api/v1/detect-nsfw` (protected, depends on `get_api_key`)

| ขั้นตอน | Error → Status |
|---|---|
| ไม่ใช่ multipart / ไม่มีฟิลด์ `image` | 400, `"No image file provided"` |
| Content-Length > 10MB | 413, `"File too large"` |
| MIME ไม่อนุญาต | 415, `"Unsupported file type"` |
| Image ไม่สามารถ decode | 422, `"Invalid image data"` |
| Model not ready | 503, `"Model not ready yet"` |
| Inference exception | 500, `"Inference failed"` (log stack ภายใน) |

**Success response:**

```json
{
  "success": true,
  "data": {
    "is_nsfw": true,
    "predictions": [
      { "label": "nsfw", "score": 0.95 },
      { "label": "normal", "score": 0.05 }
    ],
    "processing_time_ms": 23,
    "device_used": "cuda"
  }
}
```

### 7.4 Global Exception Handlers (`app/main.py`)

| Handler | Target | Output |
|---|---|---|
| `@app.exception_handler(HTTPException)` | StarletteHTTPException | envelope_error(status, exc.detail) |
| `@app.exception_handler(RequestValidationError)` | FastAPI validation | envelope_error(422, exc.body) |
| `@app.exception_handler(Exception)` | catch-all | envelope_error(500, "Internal Server Error") + log traceback |

## 12. สมมติฐาน & จุดตัดสินใจที่ต้องยืนยันกับทีม implement

1. **Health check:** ตอบ `200` ตลอดเวลา (มี `model_loaded` flag) **หรือ** `503` จนกว่าโมเดลจะพร้อม? (อาจมีผลกับ load balancer)
2. **Inference concurrency:** จำกัด 1 inference พร้อมกัน (ปกป้อง GPU) หรือ หลาย request (ถ้า VRAM พอ)
3. **CORS:** ต้องเปิดให้ domain ไหนบ้าง หรือไม่เปิดเลย (call จาก backend เท่านั้น)
4. **Auto image download ครั้งแรก (model weights ~380MB):** ระหว่าง build (pre-bake ลง image) **หรือ** mount volume ให้โหลด runtime ครั้งแรก? — แนะนำ pre-bake ลง Docker image ตอน build เพื่อ cold start เร็วขึ้น
5. **OpenAPI docs ที่ `/docs`:** เปิดไว้สำหรับ dev แต่ disable ใน production หรือไม่?
6. **Metrics:** ต้องการ Prometheus middleware (`/metrics`) หรือยังไม่ต้องใน scope นี้?

## 15. Acceptance Criteria

- [ ] `GET /health` ตอบ 200 พร้อม `device_used` (cuda/cpu)
- [ ] `POST /api/v1/detect-nsfw` ตอบ 401 เมื่อไม่มี/ผิด API key
- [ ] ปฏิเสธไฟล์ > 10MB (413)
- [ ] ปฏิเสธ MIME นอกเหนือ jpg/png/webp (415)
- [ ] ส่งภาพปกติ → `is_nsfw: false`
- [ ] ส่งภาพ NSFW → `is_nsfw: true` (threshold 0.6)
- [ ] Response มี `processing_time_ms`, `device_used`
- [ ] GPU fallback เป็น CPU อัตโนมัติเมื่อ CUDA ไม่พร้อมใช้งาน พร้อม log warning
- [ ] `docker compose up` ทำงานได้ พร้อม NVIDIA GPU passthrough
- [ ] warm start latency เหลือต่ำ โดยโหลดโมเดลครั้งเดียวที่ startup
- [ ] ไม่บล็อก async event loop ระหว่าง inference (ใช้ threadpool)
